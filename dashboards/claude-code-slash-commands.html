Claude Code · Slash Commands & Power Features

Quick reference for the commands you reach by typing / in the CLI, plus the input prefixes, permission modes, memory layers, and extension points that don’t appear in older guides.
Each command entry shows the command, what it does, a usage hint, and (where useful) a non-obvious tip. Aliases are noted in italics. Commands marked [skill] or [workflow] are bundled but otherwise behave like the regular built-ins. The page after the command tables covers input prefixes (!, @, #), plan mode, permission modes, CLAUDE.md memory, skills / sub-agents / MCP / hooks, and headless mode.

Session management

/clear — start fresh with empty context, keeping project memory.
/clear fixing-auth — names the old session in the resume picker. aliases /reset, /new
/resume — return to a previous conversation by name, ID, or picker.
/resume opens a picker; background sessions are tagged bg. alias /continue
/branch — fork the current conversation; try a new direction without losing the original.
Switches into the branch; the original is recoverable with /resume.
/rename — label the current session.
/rename auth-refactor; auto-generates a name if omitted.
/background — detach the session to run as a background agent.
/background integrate the feedback — last instruction before detaching. alias /bg
Monitor with claude agents from a fresh terminal.
/exit — quit the CLI; detaches from background sessions.
alias /quit

Model & reasoning

/model — switch the AI model.
Press s on a row to switch for current session only.
Switching mid-session re-reads history without prompt cache.
/effort — adjust reasoning depth.
Levels: low, medium, high, xhigh, max, ultracode; auto resets.
ultracode auto-orchestrates multi-agent workflows.
/fast — toggle fast mode (lower cost, slightly reduced quality).
/fast on or /fast off.

Code review & quality

/code-review — scan the diff for bugs and cleanups.
Levels low…ultra; --fix applies findings; --comment posts inline PR comments.
/simplify — review changes for cleanup opportunities and apply fixes.
Runs reuse / simplification / efficiency / abstraction agents in parallel.
Won't hunt for correctness bugs — pair with /code-review.
/review — review a GitHub PR locally.
Auto-detects PR from branch, or pass URL/number; needs gh CLI.
/security-review — scan pending changes for security issues.
Looks for injection, auth, and data-exposure problems in the diff.
/diff — interactive viewer for uncommitted changes and per-turn diffs.
←/→ switch between git diff and Claude turns; ↑/↓ browse files.

Workflow & automation

/plan — enter plan mode and lay out steps before any change.
/plan fix the auth bug — optional description.
/goal — work across turns until a condition is met.
/goal npm test succeeds; /goal clear to stop.
/batch — orchestrate large changes across the codebase in parallel.
/batch migrate src/ from Solid to React.
Decomposes into 5–30 units, one subagent per unit in a git worktree.
/fork — spawn a forked subagent that inherits the conversation.
Runs in background; result returns to the main thread.
/loop — run a prompt repeatedly on a schedule.
/loop 5m check the deploy; omit interval to self-pace.
/run — launch and drive your project's app to verify a change.
First call sets up a project-specific run skill, then re-uses it.
/verify — confirm a change by building, running, and observing.
Use when type checks and tests aren't enough proof.
/tasks — view everything running in the background.
Lists subagents, bash commands, and workflows. alias /bashes
/btw — ask a quick side question without bloating history.
/btw what's foo()'s signature?

Input prefixes & shortcuts

!cmd — run a shell command in this session.
!ls -la, !gcloud auth login; output lands in the conversation, so Claude can act on it.
Better than asking Claude to run interactive logins itself.
@path — attach a file or directory to the prompt.
@src/auth.ts what does this do?; tab-completes paths.
Cheaper than pasting code; respects .gitignore.
#note — add a memory entry inline.
#we always run tests with npm run test:unit — saved to CLAUDE.md or auto-memory.
Shift+Tab — cycle permission modes (default → accept-edits → plan).
Footer of the prompt shows the active mode.
Esc — interrupt Claude mid-response without exiting.
Double-Esc opens the message picker to edit a prior turn.
Ctrl+R — toggle verbose mode (shows full tool inputs/outputs).
Useful when a tool call surprises you.
Up arrow — recall previous prompts in this session.

Plan mode & permission modes

Plan mode — Claude explores and proposes a plan but cannot edit, write, or run mutating commands.
Enter via /plan or Shift+Tab; Claude exits with an approval prompt.
Use it for non-trivial changes — review the plan, then approve to execute.
Default mode — Claude asks before each tool the workspace hasn’t pre-approved.
Safe baseline. Approvals can be granted once, always-for-this-session, or always-for-this-project.
Accept-edits mode — auto-approve file edits; still asks for shell commands.
Good for focused refactors when you trust the diff review afterwards.
Bypass-permissions — skip all prompts (a.k.a. “YOLO mode”).
claude --dangerously-skip-permissions at launch.
Only inside a sandbox/container or a throwaway worktree.

Memory & CLAUDE.md hierarchy

./CLAUDE.md — project memory, committed to git.
Project-wide rules: build commands, conventions, “always do X / never do Y”.
./CLAUDE.local.md — project memory, gitignored.
Personal notes scoped to this repo (paths, secrets layout, your preferences).
~/.claude/CLAUDE.md — user-level memory, applies to every project.
Global rules: tone, formatting, default model, language preferences.
Auto-memory — Claude learns facts about you from conversation and stores them in ~/.claude/.../memory/.
Toggle via /memory. Each fact is a small file with frontmatter (user / feedback / project / reference).
If something keeps drifting, save it explicitly in CLAUDE.md instead of relying on auto-memory.
Discovery — Claude reads CLAUDE.md from the project root, the home dir, and any parent up the tree.
/add-dir does NOT auto-discover CLAUDE.md from the added directory.

Extending Claude Code

Skills — reusable, parameterized prompts you invoke with /skill-name.
Project skills in .claude/skills/; user skills in ~/.claude/skills/; Markdown body + YAML frontmatter.
Bundled skills (e.g. /deep-research) live alongside built-ins.
Sub-agents — specialized agents Claude can spawn for focused tasks (review, explore, plan).
Configure via /agents; project agents in .claude/agents/.
Each agent has its own context window — great for keeping the main conversation clean.
MCP servers — external integrations: GitHub, Linear, Slack, Postgres, browsers, etc.
Manage with /mcp; configure in ~/.claude.json or .mcp.json.
MCP is for third-party tools. File edits, bash, and grep are built-ins, not MCP.
Hooks — shell scripts that fire on tool events (PreToolUse, PostToolUse, UserPromptSubmit, etc.).
Defined under hooks in ~/.claude/settings.json or .claude/settings.json.
Use to enforce project policy (block bad commands, auto-format on write, run lints after edit).
Output styles — preset personas/formats for Claude’s replies.
Set in /config or via --output-style; e.g. concise, explanatory, custom.
Worktrees — isolated git worktrees so an agent can mutate files without colliding with your branch.
Auto-used by /batch and parallel sub-agents; cleaned up if no changes.

Headless mode & scripting

claude -p "…" — run a single prompt non-interactively, print the answer, exit.
claude -p "summarize today’s diff" in a script or cron job.
Combine with --output-format json for machine parsing.
claude --resume <id> — resume a specific session by ID (handy in CI).
Pair with --print to script multi-turn flows.
--model / --fallback-model — pin model at launch.
claude --model claude-opus-4-8 --fallback-model claude-sonnet-4-6.
--add-dir / --append-system-prompt — add a working dir or extend the system prompt at launch.
Useful for monorepos and bot personas in CI.
CLAUDE_CODE_NEW_INIT=1 claude — interactive first-run wizard (skills, hooks, memory).
Run once per new project to scaffold a sensible .claude/ setup.
GitHub Actions app — Claude Code as a PR reviewer/responder in CI.
Set up via /install-github-app; reacts to @claude mentions on issues and PRs.
Source: official Claude Code docs (code.claude.com/docs) and the CLI’s own /help output. Built-in commands and extension points evolve quickly — when in doubt, type /help in the CLI for the current list.
